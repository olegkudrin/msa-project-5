# Разработка дизайна модуля пакетной обработки данных

## Обоснование выбора технологии

### Анализ требований

Задача: Генерация кастомных CSV/XLSX прайс-листов для B2B-клиентов каждое утро в 6:00.

Характеристики:
- Объем данных: 5-10 тыс. строк (небольшой)
- Источник данных: База данных PostgreSQL (4 таблицы)
- Логика обработки: Простая (соединить таблицы, выгрузить в файл)
- Инфраструктура: Микросервисы в облаке
- Расписание: Простое, достаточно CRON

### Выбор и обоснование решения

Предлагается использовать **Spring Batch** (запуск с помощью K8s CronJob].

Наличие CRON-расписания:

- Можно использовать @Scheduled аннотацию Spring или внешний планировщик (K8s CronJob)
- Гибкость в выборе способа запуска

Сложность реализации логики:

- Задача не требует сложной обработки данных (запрос + выгрузка)
- Spring Batch предоставляет готовые компоненты для чтения из БД и для записи в CSV/XLS
- Потребуется минимальный объем кода для реализации логики

Ресурсоемкость:

- Легковесный Spring Boot микросервис
- Для объема данных 5-10 тыс. строк не требуется больших ресурсов

Масштабируемость:

- Легко масштабируется через Kubernetes (horizontal pod autoscaling)
- Можно запускать несколько экземпляров для разных клиентов параллельно
- Spring Batch поддерживает распределенную обработку через Remote Chunking

Интеграция с существующей инфраструктурой:

- Отличная интеграция с PostgreSQL через JDBC
- Простая генерация CSV/XLS через Apache POI

Надежность:

- Встроенная поддержка транзкций
- Retry и skip политики для обработки ошибок
- Job Repository для отслеживания выполнения задач

### Отвергнутые альтернативы

**Apache Airflow**:

Избыточен для простой задачи. Требует отдельного сервера, больше ресурсов. Подходит для сложных ETL пайплайнов с множеством зависимостей.

**K8s CronJob**:

Требует написания скрипта выгрузки с нуля. Нет встроенной поддержки транзакций и обработки ошибок. Spring Batch + K8s CronJob = лучшее сочетание, которое и предлагается.

**Spark**

Избыточен для простой задачи. Требует уже отдельного кластера. Требует внешний планировщик.

## Описание решения

Микросервис пакетной обработки (Price List Service):

1) Регулярно запускается через Kubernetes CronJob (по утрам)
2) Читает данные из рабочей базы (продукты, категории, цены)
3) Формирует файлы прайс-листов для каждого клиента
4) Сохраняет файлы прайс-листов в объектное хранилище
5) Отправляет уведомление клиентам о готовности прайсов
6) Клиенты могут забрать прайс-листы из хранилища файлов.

Предполагается, что сервис уведомлений есть готовый.
